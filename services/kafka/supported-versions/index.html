Kafka: Supported Versions

Package Versioning Scheme

  • Apache Kafka: latest stable at the time of the release, refer to the package version.
  • DC/OS: 1.10 or higher.

Packages are versioned with an a.b.c-x.y.z format, where a.b.c is the version of the DC/OS integration and x.y.z indicates the version of Apache Kafka. For example, 2.1.1-0.11.0.2 indicates version 2.1.1 of the DC/OS integration and version 0.11.0.2 of Apache Kafka.

Version Policy

The DC/OS Apache Kafka Service is engineered and tested to work with a specific release of Apache Kafka. We select stable versions of the base technology in order to promote customer success. We have selected the latest stable version of Apache Kafka for new releases.
